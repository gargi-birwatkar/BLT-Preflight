Why BLT-Preflight?

Catch security issues before they become pull requests.

Context-Aware Analysis

Evaluates issue labels, changed file patterns, and repository metadata to tailor guidance to each contribution.

Plain-Language Guidance

Every advisory includes clear, actionable recommendations with links to OWASP documentation and severity levels.

Optional Intent Capture

Contributors can share their intent to receive even more targeted guidance, fostering a collaborative security culture.

Maintainer Dashboard

Track advisory effectiveness, feedback statistics, helpfulness rates, and pattern analysis in one place.

Learning Loop

The system improves over time by collecting feedback, analyzing intent patterns, and refining guidance based on real effectiveness.

Zero-friction Integration

Drop a single GitHub Actions workflow file into your repo. No external services, no API keys, no breaking changes.

How It Works

From contribution to advisory in seconds.

  1. 1

    Contributor opens a PR or issue

    The GitHub Actions workflow triggers automatically when a pull request or issue is opened. No extra steps required from the contributor.

  2. 2

    BLT-Preflight analyses context

    The advisory engine inspects labels, changed file paths, and repository history to build a context profile for the contribution.

    Issue & PR labels
    Changed file paths
    Past patterns
  3. 3

    Advisory posted as a comment

    A structured comment appears on the PR or issue with severity-classified recommendations and links to OWASP documentation — purely advisory, never blocking.

  4. 4

    Feedback improves future guidance

    Contributors and maintainers can rate advisory helpfulness. The learning loop uses this data to refine patterns over time.

Quickstart

Get BLT-Preflight running in your repository in under 5 minutes.

1

Copy the workflow file into your repo

# .github/workflows/blt-preflight.yml
name: BLT Preflight Advisory

on:
  pull_request:
    types: [opened, synchronize, reopened]
  issues:
    types: [opened, labeled]

permissions:
  pull-requests: write
  issues: write
  contents: read

jobs:
  security-advisory:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: OWASP-BLT/BLT-Preflight@main
2

Run locally for a preview

# Clone & install
git clone https://github.com/OWASP-BLT/BLT-Preflight.git
cd BLT-Preflight
pip install -r requirements.txt

# Generate an advisory
python3 src/blt_preflight.py advise \
  --labels security,authentication \
  --files src/auth/login.py \
  --repo your-org/your-repo

# View the maintainer dashboard
python3 src/blt_preflight.py dashboard
3

Record feedback to improve guidance

python3 src/blt_preflight.py feedback \
  --pattern "Security Advisory: Authentication" \
  --helpful yes
